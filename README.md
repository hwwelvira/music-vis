# 🎵 Music Dimension Visualizer - Frontend
### 🎶 音乐流派与声学特征高维交互可视化系统 (前端 React 项目)

> **本项目定位**：音乐大屏可视化前端 React 项目独立仓库
> **技术栈**：`React 18` + `Vite` + `ECharts 5`
> **GitHub 仓库**：[Music-vis](https://github.com/hwwelvira/Music-vis)

---

## 🧭 核心功能与技术实现细节

前端系统致力于打通多维声学特征的交互与展现，包含以下四大高阶联动交互板块：

### 1. 🌊 流派结构与声学特征透视仪 (`ToggleableChordRadar`)
* **流派双层旭日图 (Sunburst)**：直观剖析多级流派层次关系。节点的亮度与透明度采用**流行度比例因子**进行缩放，低流行度流派节点自动进行最多 `55%` 的亮度淡化，主次清晰。
* **特征雷达图 (Radar Chart) 联动**：当鼠标悬停或点击旭日图的流派节点时，雷达图动态刷新，实时绘制该流派在 **可舞性、能量、声学度、情感效价、活跃度、器乐度、言语度** 7 个维度上的特征多边形。

### 2. 🌌 高维歌曲特征星云聚类与框选器 (`ScatterBrushChart`)
* **情绪散点图 (Scatter Chart)**：横轴对应 `Energy`，纵轴对应 `Valence`，在二维平面上高度聚类出歌曲的情绪色彩。
* **高阶鼠标框选 (Brush Tool)**：支持任意区域的鼠标拉框框选。框选事件在毫秒级捕获区域内所有歌曲并动态更新状态，向下分发数据。

### 3. 📑 筛选歌曲双向协同列表 (`SongTable`)
* **极速过滤响应**：无缝且快速响应散点图的框选操作，实时渲染过滤后的歌曲列表。
* **单曲反向映射高亮**：在表格中点击某首特定单曲时，上方的散点图会精确捕捉该曲坐标，以**红色圆圈扩散动画高亮包裹该歌曲节点**，提供极其灵动的双向联动。

### 4. 📅 流行音乐文化时代流动缩影 (`TimelineChart`)
* **流动时间轴 (1960-2020)**：提供手动滑动条与自动轮播演绎模式。
* **代表歌手与歌名热词双视图一键切换**。
* **100% 纯中文过滤**：内置包含近 300 个核心词的**英汉超级音乐字典**。若匹配不到任何中文翻译，则自动过滤跳过，保证大屏词云 **100% 呈现高水平的中文姿态**。
* **词频字号比例映射**：获取当前批次词频的最大/最小值，采用**归一化比例映射插值算法**，将字体大小严格控制在 `11px`（最冷门）至 `26px`（最火热）的梯度范围，强烈体现词频的高低对比。
* **无星座连线的纯文字漂移**：彻底移除了气泡背景与星座虚线，仅保留节点的物理碰撞体积。

## ⚙️ 本地部署与运行指南

### 🛠️ 环境依赖准备
请确保您的本地环境中已安装 **Node.js** (推荐 `v16` 或 `v18+`，包含 `npm`)。

### ⚛️ 启动前端本地服务
1. **克隆项目并进入项目根目录**：
   ```bash
   git clone https://github.com/hwwelvira/Music-vis.git
   cd Music-vis
   ```

2. **安装前端依赖包**：
   ```bash
   npm install
   ```

3. **启动本地开发服务器**：
   ```bash
   npm run dev
   ```
   服务器启动成功后，直接在浏览器中打开：`http://localhost:5174/` 即可开始体验！

## 🗃️ 数据源说明
* **无需运行清洗脚本**：本项目为**开箱即用**的前端可视化展示系统。经过清洗与高阶聚类的轻量级 JSON 数据源已直接内置于前端项目的 `public/data.json` 中。
* **数据处理背景**：原始数据来自 Spotify 经典歌曲数据集（包含 20MB 歌曲特征），在开发阶段已通过 Python 脚本（利用 pandas 库，包含去噪、时代歌手归档、中文超级字典词频过滤与正则归类）进行了离线预处理，大幅降低了前端运行时的计算损耗，保证极端流畅的大屏联动交互性能。
